# PartsAvenue Order Dashboard

Веб-дашборд статистики обработки заказов. Деплоится на GitHub Pages,
читает данные из приватного репо `partavenue-data`, защищён паролем.

## Файлы

- `index.html` — главная страница (lock-screen + дашборд)
- `app.js` — логика, графики, GitHub API
- `config.js` — настройки (репо/токен) — заполняется один раз
- `crypto.js` — дешифрование токена
- `encrypt.html` — утилита для шифрования токена (открывается локально)
- `demo.html` — превью дизайна с фейковыми данными
- `README.md` — этот файл

## Установка (один раз)

### 1. Зашифровать токен

1. Открой `encrypt.html` двойным кликом — откроется в браузере локально.
2. Введи свой fine-grained GitHub PAT (с правом `Contents: read & write`
   на репо `partavenue-data`).
3. Введи пароль, который будут вводить пользователи дашборда.
4. Жми «Зашифровать» → скопируй полученный blob.

### 2. Заполнить `config.js`

Открой `config.js` любым редактором и впиши:
- `REPO_OWNER` — твой GitHub-логин или название организации
- `ENCRYPTED_TOKEN` — blob из шага 1
- (по желанию) поменяй `AUTO_LOCK_SECONDS`

### 3. Залить в публичный репо

1. Создай на GitHub публичный репо `partsavenue-order-dashboard`
2. Запушь туда содержимое этой папки:
   ```bash
   cd dashboard
   git init
   git add .
   git commit -m "Initial deploy"
   git branch -M main
   git remote add origin https://github.com/<ТВОЙ_ЛОГИН>/partsavenue-order-dashboard.git
   git push -u origin main
   ```

### 4. Включить GitHub Pages

1. Зайди в репо `partsavenue-order-dashboard` → **Settings** → **Pages**
2. **Source**: Deploy from a branch → **Branch**: main / **(root)** → Save
3. Через минуту-две страница доступна по `https://<ТВОЙ_ЛОГИН>.github.io/partsavenue-order-dashboard/`
4. Делишься этим URL + паролем с партнёрами.

## Безопасность (текущий уровень)

- Токен зашифрован AES-256-GCM с ключом, выведенным из пароля
  через PBKDF2 (100k итераций, SHA-256)
- Без правильного пароля токен не расшифровывается
- Public-репо содержит ТОЛЬКО HTML/JS — никаких данных
- Все данные в приватном репо, доступны только с токеном
- Закупочные цены НЕ собираются — только цены реализации клиенту

**Уровень для тестового развёртывания.** Кто угодно с паролем может
читать и менять данные. Для боевого варианта в будущем — Cloudflare
Workers с настоящей аутентификацией.

## Что показывает дашборд

- **3 KPI-плитки**: Общий ТО / Сток / Кросс за выбранный период
- **График динамики** по дням (три линии)
- **Донат** распределения сток/кросс
- **Таблица по клиентам** с разбивкой и долями
- **Список сессий** с возможностью исключить ошибочную (двойной клик
  по галочке — флаг `included` синхронизируется в GitHub)

## Источник данных

Лаунчер (`PartsAvenue.exe`, версия 1.7.0+) после каждой «Отправить ответы»
агрегирует сессию по реестру и пушит JSON в:

```
partavenue-data/
└── data/dashboard/
    └── <YYYY-MM>.json   ← один файл на месяц
```

Дашборд читает эти файлы и рендерит.

## Обновление токена

Если токен утёк или истёк — отзови его на GitHub, создай новый,
заново прогони через `encrypt.html`, обнови `ENCRYPTED_TOKEN` в `config.js`,
запушь — пользователи продолжат заходить по тому же паролю.

Если хочешь сменить ПАРОЛЬ — нужно зашифровать токен заново с новым
паролем (старый blob не подойдёт, в нём вшит старый salt).
